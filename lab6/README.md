# Lab 6: Docker Compose

Репозиторий для 6-й лабораторной работы по курсу DevOps.
Реализованы два стека многоконтейнерных приложений с настроенным сбором метрик.

## Архитектура

### Стек 1: Приложение (Flask + Redis)
- **Flask**: Веб-приложение на Python, считающее количество входов на главную страницу. Экспортирует метрики на эндпоинт `/metrics`.
- **Redis**: In-memory база данных для хранения счетчика посещений.

### Стек 2: Мониторинг (Prometheus + Grafana)
- **Prometheus**: Система мониторинга, настроенная на скрейпинг метрик с Flask-приложения каждые 15 секунд.
- **Grafana**: Система визуализации. Источник данных (Prometheus) добавляется автоматически через provisioning.

## Как запустить

1. Склонируйте репозиторий:
   ```bash
   git clone <ссылка_на_репо>
   cd lab6
   ```

2. Поднимите все контейнеры:
   ```bash
   docker-compose up -d --build
   ```

## Как проверить

1. **Приложение**: Откройте браузер и перейдите по адресу `http://localhost:5000`. Обновляйте страницу, счетчик будет расти.
2. **Метрики**: Перейдите по адресу `http://localhost:5000/metrics`, чтобы увидеть сырые метрики Prometheus.
3. **Prometheus UI**: Откройте `http://localhost:9090`, перейдите в *Status -> Targets*. Цель `flask-app` должна быть в статусе **UP**.
4. **Grafana**: 
   - Откройте `http://localhost:3000`.
   - Перейдите в *Dashboards -> New Dashboard -> Add visualization*.
   - Выберите источник данных **Prometheus**.
   - Введите запрос: `rate(flask_http_request_total[1m])` и нажмите Run, чтобы увидеть график запросов.
